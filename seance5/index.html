Débogage

débuggage / debugging / ...

Qu'est-ce que le débogage ?

  • Inspecter un programme pour résoudre les problèmes
  • Afficher des valeurs
  • Avancer pas à pas dans les instructions
  • Modifier des valeurs à la volée
  • Reprendre l'exécution à un endroit précis'

Débogage printf

On peut déboguer avec printf

Principe

  • Afficher des valeurs intermédiaires
  • Afficher des messages à un point de l'application
  • C'EST FAUX !
  • Intrusif
  • Limité
  • Ne fait pas gagner de temps
  • Laisse des traces

GDB : GNU Debugger

  • Débuggueur ultra-répandu !
  • Interface en ligne de commandes
  • Interface graphique = ddd
  • C, C++, Java, Fortran, ...
  • Logiciel libre

Screenshot de DDD

Utilisation basique de GDB

Compiler avec -g

gcc -g foo.c -o foo.exe

Quelques commandes

  • run lancer l'exécution du programme
  • p (print) afficher la valeur d'une variable
  • b (break) créer un point d'arrêt
  • d (delete) supprimer un point d'arrêt
  • n (next) exécuter la prochaine ligne
  • s (step) exécuter la prochaine instruction
  • bt (backtrace) afficher la liste des appels précédents

Rubber Duck Debugging

Principe : expliquer le code à son canard

  • Se munir d'un canard en plastique
  • Ouvrir un fichier de code problématique
  • Expliquer le code ligne après ligne au canard
  • Répéter le processus jusqu'à ce qu'une ligne ne corresponde pas à ce que vous expliquez

Doxygen

  • Configuration de la sortie avec un fichier Doxyfile
  • Commentaires spéciaux entre /** et */ (ou /// sur une seule ligne)
  • Balises commencent par @ (ou \)
  • BaliseFonction
    @paramParamètre d'entrée d'une fonction
    @returnValeur de retour d'une fonction
    @seeRenvoie vers une autre partie du code
    @fileDécrit le fichier source
    @authorAuteur du code
